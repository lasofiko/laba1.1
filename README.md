#  Лабораторная работа №1

## Duck Typing и контрактное программирование

---

##  Цель работы

Освоить **duck typing** и **контрактное программирование** на примере разработки гибкой системы приёма задач из различных источников.

---

##  Основные задачи

-  Реализовать единый поведенческий контракт для всех источников задач
-  Использовать `typing.Protocol` и `@runtime_checkable`
-  Создать runtime-проверку соблюдения контракта
-  Разработать три типа источников: файловый, генераторный, API-заглушка
-  Обеспечить возможность добавления новых источников без изменения существующего кода

---

##  Описание проекта

Разработана платформа для приёма и обработки задач, которая может работать с задачами из разных источников:

##  Реализованные компоненты

### 1. Модель задачи (src/models/task.py)
### 2. Контракт (src/protocols/source_protocol.py)
### 3. Файловый источник (src/sources/file_source.py)
### 4. Генератор (src/sources/generate_source.py)
### 5. API-источник (src/sources/api_source.py)
### 6. Обработчик (src/processors/task_processor.py)

## Что было реализовано:
 - Duck typing — источники не наследуют контракт, но имеют нужные методы

 - Контрактное программирование — через Protocol задан единый интерфейс

 - Runtime проверка — isinstance(source, TaskSource) защищает от ошибок

 - Три типа источников — файл, генератор, API-заглушка

 - Гибкость архитектуры — новые источники подключаются без изменения кода

 - Тесты — 11 тестов, покрывающих всю функциональность

## Навыки, освоенные в работе:
- Использование typing.Protocol для создания контрактов

- Применение декоратора @runtime_checkable

- Понимание duck typing и его преимуществ

- Разработка расширяемой архитектуры

- Написание тестов с pytest
